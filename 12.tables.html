jQuery Mobile - Tables

Responsive Tables

Responsive design is useful when you want the content of your mobile web page to respond to the users device, such as its screen size and orientation.

With a simple class name, jQuery Mobile is aware of the users available screen size and automatically resizes itself to show content that is relevent for that particular user.

Responsive tables allow us to display a large set of tabular data that will look attractive for both mobiles and desktop.

There are two types of responsive tables: reflow and column toggle.

Reflow Table

The reflow mode positions the table data horizontally until it reaches a minimum size, then all rows are grouped together vertically.

Create a table, add the data-role="table" attribute, and a class of "ui-responsive" on the table-element.

Example: reflow table

Name Date
Joe 1979
Mike 1977

For the responsive table to work correctly, you MUST include the thead-element (for th) and the tbody-element (for td).

Do NOT use row-span or col-span attributes; they are not supported.

Column Toggle Table

The "column toggle" mode will hide columns when there is not enough width to display the data.

To create a column toggle table, add the following to the table element:


			/table data-role="table" data-mode="columntoggle" class="ui-responsive" id="myTable"/

By default, jQuery Mobile will hide columns from the right side of the table. However, you are allowed to specify which column that should be hidden or shown in a specific order. Add the data-priority attribute to the table's header () and specify a number between 1 (highest priority) to 6 (lowest priority) for each th-element:

If you don't specify a priority for a column, the column will be persistent and not available for hiding.

Example: column toggle table

Name Date Color ID
Jack 1989 Black #45
John 1943 Blue #33
Lynda 1967 Brown #72
Lacy 1988 Green #63

To change the toggle table button text, use the data-column-btn-text attribute.
