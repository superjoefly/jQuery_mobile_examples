jQuery Mobile - Navbars

A navigation bar consists of a group of links that are aligned horizontally, typically within a header or footer.

The navbar is coded as a unordered list of links wrapped inside a div-element that has the data-role="navbar" attribute.

Example: navbar

  • Link 3

By default, links inside a navigation bar will automatically turn into a button.

The buttons are, by default, as wide as its content. Use an unordered list to divide the buttons equally: 1 button takes 100% of the width, 2 buttons share 50% each, 3 buttons 33.3%, etc. However, if you specify more than 5 buttons in the navbar, it will wrap to multiple lines.

Icons in Navigation Buttons

To add an icon to your naviagtion button, use the data-icon attribute.

The data-icon attribute uses the same values as the CSS classes specified in the icons chapter. The only difference is that, instead of specifying class="ui-icon-value", you specify the attribute of data-icon="value".

Positioning Icons

Just like with the "ui-btn-icon-position" class (specified in the "Icons" chapter), you can choose where the icon should be positioned in the navigation button: top, right, bottom or left.

The icon position is set on the navbar container - it is not possible to position each individual button link. Use the data-iconpos attribute to specify the position:

Active Buttons

When a link in the navbar is tapped/clicked, it gets the selected (pressed down) look.

To achieve this without having to tap the link, use the "ui-btn-active" class.
