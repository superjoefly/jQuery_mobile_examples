jQuery Mobile - Forms

jQuery Mobile automatically styles HTML forms to make them look engaging and touch friendly.

jQuery Mobile Form Structure

jQuery Mobile uses CSS to style HTML form elements, making them attractive and easy to use.

In jQuery mobile you can use the following form controls:

  • Text Inputs
  • Search Inputs
  • Radio Buttons
  • Checkboxes
  • Select Menus
  • Sliders
  • Flip Toggle Switches

When working with jQuery Mobile forms, you should know:

  • The form-element must have a method and an action attribute.
  • Each form-element must have a unique "id" attribute. The id must be unique accross the pages in the site. jQuery's single-page navigation model allows many different "pages" to be present at the same time.
  • Each form element must have a label. Set the for attribute of the label to match the id of the element.

Tip: use a placeholder to specify a shor hint that describes the expected value of an input field.

Example: form with placeholder

Tip: To hide the label, use the "ui-hidden-accessible" class. This is often used when you want the element's placeholder attribute to serve as the label.

Example: form with hidden label

Tip: if you want a "clear" button (an X icon on the right side of the input field that clears the contents of the field), add the data-clear-btn="true" attribute.

Example: form with "clear" button

The "clear button" can be added on any /input element, except for /textarea. Search fields have this attribute set to "true" as default - to change it, simply specify data-clear-btn="false".

jQuery Mobile Form Buttons

Buttons in forms are coded with standard HTML input-elements (button, reset, submit). They are automatically styled, making them attractive and easy-to-use on both mobile devices and desktop computers:

Example: mobile form buttons

To additionally style an input-button, use any of the data-* attributes listed in the table below:

Attribute Value Description
data-corners true | false specifies rounded corners
data-icon see reference specifies icon
data-iconpos left | right | top | bottom | notext specifies position of icon
data-inline true | false specifies if button is inline or not
data-mini true | false specifies whether button should be small or not
data-shadow true | false specifies shadow or no shadow

Field Containers

To make labels and forms loop proper on wider screens, wrap a div-element or fieldset-element with the "ui-field-contain" class around the label/form element.

Example: field container

On larger screens (greater than 480px) the label will be on the same line as the form control.

Tip: To prevent jQuery Mobile from automatically styling tappable/clickable elements, use the data-role="none" attribute.

Example: form without styling

jQuery Mobile will automatically handle the form submission via AJAX, and will attempt to integrate the server response into the DOM of the application.
